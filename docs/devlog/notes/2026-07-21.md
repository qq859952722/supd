## 8. 服务/扩展 user 字段接入进程启动（重大 bug 修复）

**背景**：服务 `service.yaml` 的 `user` 字段未接入进程启动，服务全部以 supd 用户运行。规格 §2.2.13 要求按指定用户权限启动。

**根因**：
1. `process.go` 的 `StartProcess` 支持 `credential` 参数，但 4 处调用点全传 `nil`
2. 服务级扩展 `ServiceUser` 未传播（dispatcher 未填充 `TriggerContext.ServiceUser`）
3. 无统一身份解析层（core 不能 import extension，会循环依赖）

**实现**：
- 新建 `internal/identity` 共享叶子包（`LookupUserGroups`/`BuildCredential`/`GetCurrentUser`）
- 新建 `internal/core/credential.go`：`ResolveServiceCredential` + `StartServiceProcess`
- 4 处 `StartProcess` 调用改为 `StartServiceProcess`，传入 `svcConfig.User`
- `dispatcher.go`：`matchedExtension` 增加 `serviceUser` 字段，服务级扩展填充 `svcEntry.Config.User`，全局扩展传空
- `run_as.go`：用户查找失败返回含解决方法的详细错误
- `service_ops.go`：`errors.As` 识别 `*ServiceError` → HTTP 422

**语义差异**：服务（严格）非 root 切换→拒绝 422；扩展（宽松）→警告+当前用户运行。

**重大 bug 修复（代码审计发现）**：全局扩展被 `service_lifecycle` 触发时错误继承服务 user，违反规格。修复：全局扩展 `TriggerContext.ServiceUser` 始终为空。

**运行测试**：7 场景全通过（不存在用户 422、空 user 继承、非 root 切 root 422、扩展 run_as 不存在 failed、服务级扩展继承服务 user 等）。
