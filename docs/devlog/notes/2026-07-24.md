### 关键技术点
- **tjs 递归文件操作**：`tjs.readDir` 返回数组（元素可能是 string 或 {name} 对象，需 `typeof entry === 'string' ? entry : entry.name` 兼容）；递归判断目录用 `try { await tjs.readDir(path); isDir=true } catch {}`
- **tjs.chown 不支持递归**：整目录改属主必须用 shell `chown -R` 或手动递归
- **tjs.spawn stdout 是 ReadableStream**：不能直接当字符串，必须 `getReader()` 流式读取
- **Transmission RPC CSRF**：首次请求返回 409 + `X-Transmission-Session-Id` 头，携带该头重试即成功
- **服务直接启动二进制**：避免 start.sh 包装脚本，二进制更新/目录初始化交给 on_demand 扩展（run_as: root），service.yaml 的 command 只负责启动常驻进程
- **权限协调**：服务 `user: nobody` 降权运行；扩展 `run_as: root` 提权执行 chown/安装；扩展安装后 `chown -R nobody:nobody` 确保服务进程可读写

## 3. Transmission 端口探测 BUG 修复（Yama LSM ptrace_scope 降级方案）

### 触发原因
用户发现 192.168.31.188:7979 的 supd 服务中 transmission 服务没有探测到端口（9091/51413），但端口实际对外可达。

### 根因分析

**核心问题**：Linux Yama LSM `ptrace_scope=1`（restricted 模式）阻止 root 进程读取非 root 用户进程的 `/proc/<pid>/fd/` symlink。

**证据链**：
1. 容器内 `cat /proc/sys/kernel/yama/ptrace_scope` → **1**（restricted）
2. Transmission 进程（PID 27）以 `user: nobody`（UID 65534）运行
3. `readlink /proc/27/fd/3` → **Permission denied**（空输出）
4. 对比 root 进程（PID 31, dropbear）`readlink /proc/31/fd/3` → **正常返回** `socket:[502168949]`
5. `/proc/net/tcp` 中 UID=65534 的行清晰可见（9091/51413），但 supd 端口采集代码因无法通过 inode 匹配而返回空

**代码路径**：`collectSocketInodes(pid)` → `os.Readlink(/proc/27/fd/*)` 全部 Permission denied → 返回空 inode map → `collectProcessPorts` 返回 nil → API 无 ports 字段

**为什么 CPU/内存能采集**：`collectProcessResources` 只读 `/proc/<pid>/stat` 和 `/proc/<pid>/status`（不受 Yama 限制），而端口采集需 `readlink /proc/<pid>/fd/*`（受限）。

### 修复方案

**UID 降级方案**：当 inode 精确匹配失败（`collectSocketInodes` 返回空）时，自动降级到通过进程 UID 在 `/proc/net/{tcp,tcp6,udp,udp6}` 中匹配端口。

- `/proc/<pid>/status` 中的 `Uid:` 行不受 Yama 限制，可正常读取
- `/proc/net/` 文件中的每行含 `uid` 字段，可按 UID 匹配
- UID 匹配精度低于 inode 匹配（同 UID 的所有进程端口都会被匹配），但服务通常独占一个 UID（nobody/特定用户），精度足够

新增三个函数：
1. `getProcessUID(pid)` — 从 `/proc/<pid>/status` 读取真实 UID
2. `getChildProcessUIDs(pid)` — 收集子进程 UID 列表
3. `matchNetSocketsByUID(proto, uids)` — 在 `/proc/net/{proto}` 中按 UID 匹配端口

修改两个函数：
1. `collectProcessPorts(pid)` — inode 为空时自动降级到 UID 匹配
2. `collectProcessPortsByCommand(cmdPattern)` — 调用 collectProcessPorts（内含 UID 降级）
