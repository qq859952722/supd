## 4. Dropbear SSH 第 3 版方案 + 服务 env.yaml 加载 BUG 修复

**用户反馈**：dropbear 不应在 compose 配置 SSH_PUBLIC_KEY；服务默认不自动启动；可通过服务环境变量配置免认证。

**BUG 修复：服务进程加载 services/<svc>/env.yaml（规格 §2.2.4）**
- 根因：`bootstrap.go` 两处服务进程启动用 `os.Environ()`，未加载服务级 env.yaml
- 修复：新建 `internal/core/service_env.go`，`buildServiceProcessEnv` 合并 3 层 env（os.Environ → 全局 env 文件 → 服务 env.yaml）；bootstrap.go 两处替换
- 新增 7 个单元测试

**Dropbear 第 3 版**：
- dropbear-ssh 改为 `autostart: false` + run.sh 脚本（自动选认证模式）+ env.yaml（`SSH_PUBLIC_KEY` + `DROPBEAR_PORT`）
- 删除 setup-ssh-keys 扩展（公钥配置职责移至 run.sh）
- docker-compose 移除 SSH_PUBLIC_KEY 环境变量
- 认证模式：SSH_PUBLIC_KEY 非空→公钥认证；空→免认证（`dropbear -B` + `passwd -d`）

**文件**：`service_env.go`（新）、`service_env_test.go`（新）、`bootstrap.go`、`docker-compose.yml`、`init_examples.go`、`init.go`、`init_test.go`、`SKILL.md`

## 5. 系统审计修复 3 项规格偏差

**触发**：用户要求审计是否还有类似 env.yaml 未加载的重大偏差。

**偏差 #1（🔴 重大）：script readiness 未继承服务环境变量**（规格 §2.2.3）
- 根因：`readiness_script.go` 的 `Check()` 未设 `cmd.Env`
- 修复：`NewReadinessChecker(cfg, dir, env)` 增加 env 参数；scriptChecker 增加 env 字段 + `cmd.Env = s.env`
- 附带修复：`service_operator.go` 2 处 `os.Environ()` → `BuildServiceProcessEnv`（API 启动的服务此前也未加载 env.yaml）

**偏差 #2（🟠 重要）：on_failure 阶段 SUPD_SERVICE_PID 未注入**（规格 §2.2.5）
- 根因：`trigger_lifecycle.go` 的 `OnFailure` 未设置 `DispatchRequest.ServicePID`
- 修复：`OnFailure` 签名增加 `servicePID int`；传播链：`bootstrap.go` → `run.go` → `service_operator.go` 传入 `proc.PID()`

**偏差 #3（🟡 轻微）：cronScheduler.Stop() 无超时**（规格 §2.8.1）
- 修复：`Stop(ctx context.Context)` 用 select 竞争 `stopCtx.Done()` 与 `ctx.Done()`

**签名变更（后续新增调用处需注意）**：
- `NewReadinessChecker(cfg, dir, env)` — 3 参数
- `OnFailure`/`OnServiceFailure` — 增加 `servicePID int`
- `CronScheduler.Stop(ctx)` — 带 context
